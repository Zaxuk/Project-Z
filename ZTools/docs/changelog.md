#### API 分页优化
- **正确的分页参数**: 发现禅道 8.x `my-story-assignedTo` API 的正确使用方式
  - URL 格式: `/my-story-assignedTo-id_desc-{total}-{limit}-{page}.json`

  - `limit`: 分页大小（最大 200）
  - `page`: 页码（从 1 开始）
- **自动分页**: 实现循环查询，直到返回数据为空，最多查询 10 页（2000 条需求）

#### 性能优化
- **先过滤后检查**: 优化未创建任务需求的查询逻辑
  - 先按阶段过滤（已计划/已立项）
  - 再检查任务创建情况
  - 减少 API 调用次数，提高查询效率

- `GET /task-view-[taskID].json` - 查看任务

## [2026-02-28] v0.3.3 - 任务查询默认过滤

### 新增功能

#### 默认状态过滤
- **默认查询**: 当用户未指定状态时，默认只查询等待（wait）和进行中（doing）的任务
- **状态过滤支持**: 用户可以通过自然语言指定状态进行过滤
  - "查看我已完成的任务" → 查询已完成（done）的任务
  - "查看进行中的任务" → 查询进行中（doing）的任务
  - "查看所有任务" → 查询所有状态的任务

#### 技术实现
- 修改 `skill.py` 中的 `_handle_query_tasks` 方法
- 当 `status` 为 `None` 时，分别查询 `wait` 和 `doing` 状态的任务并合并结果
- 当 `status` 有值时，按指定状态查询

#### 支持的过滤指令
```
- "查看我的任务" → 默认查询等待和进行中
- "查看等待的任务" → 查询 wait 状态
- "查看进行中的任务" → 查询 doing 状态
- "查看已完成的任务" → 查询 done 状态
- "查看已关闭的任务" → 查询 closed 状态
```
