## [2026-03-03] v0.5.0 - 需求拆解流程完善

### 新增功能

#### 非交互模式支持
- **命令行参数支持**: 支持通过命令行参数直接传入任务创建信息
  - `--grade` / `-g`: 需求等级 (A-/A/A+/A++/B)
  - `--priority` / `-p`: 需求优先级 (紧急/非紧急)
  - `--online-time` / `-o`: 需求上线时间
  - `--assigned-to` / `-a`: 任务执行人
  - `--hours`: 任务时长
  - `--deadline` / `-d`: 任务截至时间
  - 示例: `python skill.py 拆解需求#11754 -g A+ -o 下下周周一 -a zhuxu --hours 8`

#### 项目选择功能
- **智能项目选择**: 支持配置默认项目或让用户选择
  - 配置文件支持 `default_project_name` 设置默认项目名称（支持部分匹配）
  - 未配置时自动显示未完成的项目列表供选择
  - 自动过滤已完成(done)和已关闭(closed)的项目
  - 支持部分匹配：配置 "都江堰" 可匹配 "都江堰项目"

#### 需求评审功能
- **自动评审**: 变更需求标题后自动调用评审接口
  - 将需求状态从 "已变更(changed)" 改回 "已激活(active)"
  - 评审结果设为 "通过(pass)"
  - 自动传递任务执行人和预计工时到需求
  - 如果需求状态已经是 active，自动跳过评审

### 改进优化

#### 需求标题处理
- **修复标题重复问题**: 多次拆解需求时避免标题内容重复
- **智能标签提取**: 正确提取和保留原标题中的标签

#### API 适配
- **完善任务创建**: 使用正确的字段名 `assignedTo[]` 和必要的表单参数
- **添加评审接口**: 实现 `review_story` 方法支持需求评审

### 配置文件更新
- `config/settings.yaml`:
  - 将 `default_project_id` 改为 `default_project_name`，支持名称匹配
  - 添加注释说明支持部分匹配功能
