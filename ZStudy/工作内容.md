## 二、组织架构定位

点三电商OMS采用**中台化架构**，分为三大领域：

```
┌─────────────────────────────────────────────────────────────┐
│                    点三电商OMS产品层                          │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ 电商中台  │  │ 业务中台  │  │ 基础中台  │  │ ERP对接  │    │
│  │(另一团队) │  │(我的团队) │  │(共建)    │  │(我的团队) │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
└─────────────────────────────────────────────────────────────┘
```

- **电商中台**：60+电商平台对接（淘宝、京东、拼多多等）
- **业务中台**：进销存核心系统（我的团队负责）
- **基础中台**：用户、商品、店铺等基础数据（共建）
- **ERP对接**：金蝶、用友等主流ERP系统（我的团队负责）

## 三、核心工作职责

### 3.1 架构设计（30%）

| 职责 | 具体内容 |
|-----|---------|
| 中台架构规划 | 业务中台整体架构设计、领域划分、服务拆分 |
| 技术选型 | 微服务架构、数据库设计、中间件选型 |
| 架构治理 | 代码重构、技术债务清理、性能优化 |

### 3.2 系统建设（40%）

#### 业务中台核心领域

| 领域 | 核心功能 |
|-----|---------|
| 订单域 | 订单生命周期管理、订单策略引擎（自动合并/拆分） |
| 库存域 | 库存计算引擎、库存同步、防超卖策略 |
| 采购域 | 采购订单、供应商管理 |
| 仓储域 | 出入库管理、三方仓对接 |
| 售后域 | 退换货、退款、售后策略 |

#### ERP对接平台

| ERP系统 | 对接内容 |
|--------|---------|
| 金蝶KIS云 | 订单、库存、财务数据互通 |
| 金蝶星空ERP | 全渠道解决方案对接 |
| 金蝶精斗云ERP | 租户自动开通 |
| 用友智邦ERP | 业务数据同步 |

### 3.3 框架能力建设（20%）

自主研发**业务框架化能力**，沉淀可复用组件：

| 框架 | 功能描述 | 应用场景 |
|-----|---------|---------|
| 业务单据锁框架 | 注解化分布式锁，自动加锁与释放 | 并发控制 |
| 业务扩展属性框架 | 热点数据自动落扩展表 | 业务灵活扩展 |
| 单位换算框架 | 注解驱动单位换算/反算 | 多单位业务 |
| 业务扩展值框架 | 扩展表数据自动回注业务对象 | 数据组装 |
| 业务单据归档框架 | 历史数据自动归档 | 性能优化 |
| 状态机框架 | 订单状态流转标准化 | 状态管理 |

### 3.4 工程化与团队管理（10%）

| 职责 | 具体内容 |
|-----|---------|
| CI/CD建设 | Docker + Drone搭建自动化构建部署平台 |
| 环境管理 | hotfix分支测试服务动态切换 |
| 技术规范 | 代码规范制定、Code Review机制 |
| 团队培养 | 技术分享、能力提升 |
