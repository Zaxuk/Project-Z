## 六、面试核心话术（持续更新）

### 事务失效场景
> "Spring 事务同类内部调用会有失效问题。比如 methodA 调用 methodB，两个方法都有 @Transactional，A 的事务是生效的，但 B 的事务会失效，因为内部调用绕过了代理对象。我在项目中遇到过这种情况，当时是需要 B 方法独立事务记录日志。解决方案是注入自身代理对象再调用，这样 B 的事务就能生效了。"

### 事务传播行为
| 传播行为 | 含义 | 使用场景 |
|---------|------|---------|
| REQUIRED（默认） | 有事务加入，无事务新建 | 大多数业务场景 |
| SUPPORTS | 有事务加入，无事务非事务执行 | 查询类方法 |
| MANDATORY | 必须有事务，否则抛异常 | 强制事务场景 |
| REQUIRES_NEW | 挂起当前事务，新建独立事务 | 日志记录等独立操作 |
| NOT_SUPPORTED | 挂起当前事务，非事务执行 | 不需要事务的方法 |
| NEVER | 必须无事务，否则抛异常 | 纯查询场景 |
| NESTED | 嵌套事务，可独立回滚 | 复杂业务流程 |

### Spring IOC 流程
```
Spring IOC 核心流程：
1. 读取配置类，解析@ComponentScan扫描路径
2. 扫描包路径，识别@Component/@Service等注解的类
3. 为每个Bean生成BeanDefinition（Bean的元数据）
4. 调用BeanFactoryPostProcessor（修改Bean定义）
5. 实例化Bean（反射调用构造方法）
6. 填充依赖（@Autowired/@Value注入）
7. 调用Aware接口方法（获取容器资源）
8. 调用BeanPostProcessor.postProcessBeforeInitialization
9. 调用@PostConstruct/init-method
10. 调用BeanPostProcessor.postProcessAfterInitialization
11. Bean就绪，放入单例池
```
